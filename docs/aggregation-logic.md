# 数据聚合逻辑详解

## 聚合维度
**按 站点日期 + 国家 聚合**

## 过滤规则（关键！）

### 基础过滤（应用于所有统计）
```python
订单状态 != 'Canceled' AND 换货订单 != '是'
```

### 销售额和优惠券的额外过滤
```python
订单状态 != 'Canceled' AND 换货订单 != '是' AND 是否退货 != '是'
```

## 字段计算逻辑

### 从 order_list 计算：

1. **货币**
   - 取值：`订单币种`（同一国家应该一致）
   - 逻辑：`first`

2. **总销量**
   - 来源：`数量` 列
   - 过滤：基础过滤
   - 逻辑：求和

3. **FBM订单**
   - 来源：`数量` 列
   - 过滤：基础过滤 + `订单类型 == 'MFN'`
   - 逻辑：求和

4. **FBA订单**
   - 来源：`数量` 列
   - 过滤：基础过滤 + `订单类型 == 'AFN'`
   - 逻辑：求和

5. **总销售额**
   - 来源：`单价` 列
   - 过滤：基础过滤 + 是否退货过滤
   - 逻辑：求和

6. **优惠券订单数**
   - 来源：订单计数
   - 过滤：基础过滤 + 是否退货过滤 + `促销编码` 不为空
   - 逻辑：计数

### 从 order_profit 计算：

7. **货币**（用于填补 order_list 中缺失的货币）
   - 来源：`币种` 列
   - 逻辑：`first`

8. **广告单**
   - 来源：`广告销量` 列
   - 逻辑：求和

9. **总广告花费**
   - 来源：`广告花费` 列
   - 逻辑：绝对值求和

10. **今日退款数量**
    - 来源：`退款量` 列
    - 逻辑：求和

11. **今日退款金额**
    - 来源：`退款金额` 列
    - 逻辑：绝对值求和

## 数据合并逻辑

```python
# 1. 从 order_list 聚合
list_agg = valid_orders.groupby(['站点日期', '国家']).agg(...)

# 2. 从 order_profit 聚合
profit_agg = profit_df.groupby(['站点日期', '国家']).agg(...)

# 3. 外连接合并（outer join）
result = list_agg.join(profit_agg, how='outer')

# 4. 货币填补：优先使用 list 的货币，如果为空则用 profit 的
result['货币'] = result['货币'].fillna(result['货币_profit'])

# 5. 数字列填充 0
result['总销量'] = result['总销量'].fillna(0).astype(int)
# ... 其他列同理
```

## 特殊情况处理

### 情况1：order_list 有数据，order_profit 无数据
- 结果：广告单、总广告花费、今日退款数量、今日退款金额 = 0

### 情况2：order_profit 有数据，order_list 无数据
- 结果：总销量、FBM订单、FBA订单、总销售额、优惠券订单数 = 0
- 货币：使用 profit 表的币种
- 示例：测试数据中的加拿大（profit 有数据，list 无有效订单）

## 输出格式

CSV 文件，UTF-8-BOM 编码，列顺序：
```
站点日期,国家,货币,总销量,FBM订单,FBA订单,广告单,总销售额,优惠券订单数,总广告花费,今日退款数量,今日退款金额
```

数据类型：
- 站点日期：字符串（ISO 格式）
- 国家：字符串
- 货币：字符串（CAD, USD 等）
- 整数列：总销量、FBM订单、FBA订单、广告单、优惠券订单数、今日退款数量
- 浮点数列：总销售额、总广告花费、今日退款金额
